## Changes Since v0.3.1

### Runtime changes reviewed

- `Configuration/Icons.php` — new file. Registers the backend module SVG icon using the
  TYPO3 12+ `Icons.php` convention. Contains only a PHP return array with class references
  and a resource path. No executable logic, no user input. No security impact.

- `ext_localconf.php` — removed `IconRegistry::registerIcon()` call and the three
  `use` imports for `IconRegistry`, `SvgIconProvider`, `GeneralUtility`. This is a
  reduction of the code surface; no security impact.

- `composer.json` — `typo3/cms-*` constraints widened to `^13.4 || ^14.0`. No new
  runtime dependencies introduced. `b13/make` removed from require-dev (local scaffolding
  tool, never executed in production). No security impact.

- `ext_emconf.php` — TYPO3 version constraint updated to `13.4.0-14.99.99`. Metadata
  only, no security impact.

### CI / tooling changes (no runtime impact)

- `.gitlab-ci.yml` — extended with PHP 8.5 and TYPO3 14 test matrix jobs. CI-only,
  no runtime impact.
- `.git/hooks/pre-commit`, `commit-msg` — fixed to call GrumPHP via `ddev exec`.
  Git hooks only, no runtime impact.
- `.claude/` — project manager agent, guidelines, commands added. Development tooling
  only.

### Security-relevant finding: none

No changes affect the extension's runtime security posture. The icon registration
refactor reduces code surface. No new attack vectors introduced.

## OWASP Checklist

### A01 — Broken Access Control
- [x] No endpoint is accessible without the consumer implementing authentication
- [x] Consumer security responsibilities are documented in the extension docs
- [x] No TYPO3 page access restrictions are inadvertently bypassed

**Notes:** Unchanged since v0.3.0. Extension deliberately delegates auth to the consumer.

### A03 — Injection
- [x] All URL parameters are type-coerced before being passed to consumer methods
- [x] No raw URL parameter values are used in TYPO3 queries or system calls within the extension
- [x] `InvalidParameterException` (400) is thrown on coercion failure

**Notes:** No changes to parameter handling in this release.

### A04 — Insecure Design
- [x] Extension does not handle auth, authz, or rate limiting (by design — documented)
- [x] Consumer responsibility documentation is present and up to date
- [x] No new features implicitly assume security guarantees not provided by the extension

**Notes:** No new features in this release.

### A05 — Security Misconfiguration
- [x] Base path validation enforces correct format (`/name/`)
- [x] Route enhancer configuration is validated
- [x] Default configuration exposes minimum necessary surface

**Notes:** No configuration changes in this release.

### A06 — Vulnerable Components
- [x] `composer audit` output: no known vulnerabilities
- [x] All dependencies are justified and minimized
- [x] No new unjustified dependencies added since last audit

**Notes:** `b13/make` removed. No new dependencies added.

### A09 — Security Logging
- [x] Invalid parameter type triggers a WARNING log entry
- [x] Endpoint not found triggers a WARNING log entry
- [x] Log channel `simple_rest_api` is used consistently

**Notes:** No changes to logging in this release.
