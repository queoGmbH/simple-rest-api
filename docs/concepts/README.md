# Feature Concepts

This directory contains concept documents for features and significant changes to the simple_rest_api extension.

A concept document is written **before implementation begins**. It describes what is being built, why, and how — serving as a reference during and after development.

## When to write a concept document

- For any new feature that affects the public API
- For significant refactorings that change the extension's architecture
- When multiple implementation approaches exist and a decision must be made
- When the Architect agent is involved in planning a feature

## Lifecycle

Concepts are **permanent references** — they are not deleted after implementation. They serve as historical context for why a feature was built the way it was.

After implementation:
- Update the **Status** to `Implemented`
- Add a reference to the relevant ADR(s) if architectural decisions were made

## Format

Files are named `feature-short-name.md` (e.g. `configurable-base-path.md`).

Use the template in `_template.md`.
